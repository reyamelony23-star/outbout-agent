# Moiboo Outbound — Web App

A full-stack Flask app that wraps the outbound sales pipeline (Google Maps
prospecting → personalized pitch decks → Gmail outreach) into a single
dashboard, with a Claude-powered chat assistant.

## Features

- **Dashboard** with live stats pulled from Google Sheets
- **Prospects** table with per-row "Generate Deck" and "Send Outreach" buttons
- **Search bar** that triggers an Apify Google Maps run and writes results to Sheets
- **Pitch decks** generated by python-pptx, with copy written by Claude
- **Cold outreach** via Gmail SMTP, with copy written by Claude
- **Chat assistant** (bottom-right bubble) — natural language → Claude tool use →
  the right backend action (`search`, `generate_deck`, `send_outreach`)
- **Auth** with two roles: `admin` sees everything, `client` sees only their own rows

## File structure

```
outbound-agent/
  app.py                  # Flask factory + login manager
  config.py               # env-loaded API keys + Flask secret
  auth.py                 # /login, /logout, Flask-Login User
  chat.py                 # Claude tool-use router for the chat bubble
  scraper.py              # Apify Google Maps scraper
  sheets.py               # Google Sheets backend (used as the DB)
  deck_generator.py       # python-pptx + Claude personalized decks
  outreach.py             # Gmail SMTP + Claude email copy
  routes/
    main.py               # /dashboard, /prospects, /campaigns, /decks, /settings
    actions.py            # /search, /generate-deck, /send-outreach, /chat
  templates/              # Jinja2 templates
  static/
    css/style.css         # Moiboo branding (blue #0099CC, navy #1a1a2e)
    js/app.js, chat.js    # row actions + chat bubble
  requirements.txt
  .env                    # API keys (gitignored)
  credentials.json        # Google service account key (gitignored)
```

## Setup

1. **Install deps:**
   ```
   python3 -m venv .venv && source .venv/bin/activate
   pip install -r requirements.txt
   ```
2. **Fill in `.env`** (already created — verify it has all keys):
   - `ANTHROPIC_API_KEY`, `APIFY_API_KEY`, `GOOGLE_SHEETS_ID`
   - `GMAIL_ADDRESS`, `GMAIL_APP_PASSWORD`
   - `GOOGLE_SERVICE_ACCOUNT_FILE=credentials.json`
   - `ADMIN_PASSWORD=<your-choice>` (used to bootstrap the first admin user)
   - `FLASK_SECRET_KEY=<random-string>` (any long random value)
3. **Drop your Google service account JSON** at `./credentials.json`.
   The service account email must be shared on the sheet as an **Editor**.
4. **Run it:**
   ```
   python app.py
   ```
5. Open <http://127.0.0.1:5000>. On first request the app creates the `Users`
   and `Prospects` tabs and seeds an admin user (`GMAIL_ADDRESS` /
   `ADMIN_PASSWORD`). Sign in with those credentials.

## Sheet schema

The app uses Google Sheets as its database. Two tabs:

**Users** — `Email | Password Hash | Role | Name | Created At`

**Prospects** — `Business Name | Address | Phone | Website | Email | Rating |
Review Count | Lead Score | Search Query | Status | Deck Path | Last Contacted |
Owner Email | Notes`

## Chat assistant — what it understands

The chat bubble routes via Claude tool use to one of three actions. Examples:

| You type                                       | Action                  |
| ---------------------------------------------- | ----------------------- |
| `find me workshops in Jurong`                  | `search_prospects`      |
| `generate deck for ABC Auto`                   | `generate_deck`         |
| `send outreach to all new prospects`           | `send_outreach all_new` |
| `email XYZ Workshop`                           | `send_outreach <name>`  |

`all_new` is capped at 5 emails per chat invocation to avoid surprises.

## Notes

- **Sheets-as-DB is fine for demos.** For production, swap `sheets.py` for
  Postgres/SQLite. The shape of the helpers stays the same.
- **Claude model:** every Claude call uses `claude-opus-4-7`. System prompts
  are cached (`cache_control: ephemeral`).
- **First admin password** lives in `ADMIN_PASSWORD`. Rotate it after first
  login by editing the `Password Hash` cell directly in the sheet, or by
  adding a settings page that does the same via `werkzeug.security`.
